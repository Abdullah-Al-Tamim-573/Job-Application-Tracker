1. What is the difference between getElementById, getElementsByClassName, and querySelector / querySelectorAll?

Ans: getElementById, getElementsByClassName, querySelector, querySelectorAll হলো DOM-এর কিছু মেথড, যার মাধ্যমে আমরা HTML এলিমেন্টগুলোকে ধরতে পারি। এরপর সেই অনুযায়ী এলিমেন্টের কালার পরিবর্তন, ট্যাগ অ্যাড বা রিমুভ এবং আপগ্রেড করার মতো কাজগুলো করতে পারি।

getElementById:

এটি কেবল একটি নির্দিষ্ট এলিমেন্টকে ধরতে সহায়তা করে এবং HTML-এ থাকা id দিয়ে এটি ধরতে হয়।

এটি সরাসরি ওই এলিমেন্টটিকেই রিটার্ন করে।

যদি কিছু খুঁজে না পায়, তবে এটি null রিটার্ন করে।

getElementsByClassName:

এটি একসাথে একাধিক উপাদান বা এলিমেন্টকে ধরতে সহায়তা করে।

এটি মূলত একটি HTMLCollection রিটার্ন করে, যা দেখতে অ্যারের মতো কিন্তু আসলে অ্যারে নয় (Array-like object)। তাই এতে সরাসরি অ্যারের মেথডগুলো কাজ করে না; আমাদের লুপ চালিয়ে এটি অ্যাক্সেস করতে হয়।

কিছু খুঁজে না পেলে এটি একটি খালি (empty) HTMLCollection রিটার্ন করে।

এটি ডাইনামিক।

querySelector:

এটি CSS সিলেক্টরের মতোই কাজ করে।

এর ভেতরে ক্লাস নেম দিলে . দিতে হয়, আইডি দিলে # দিতে হয় এবং ট্যাগ নেম থাকলে শুধু ট্যাগ নেম দিলেই হয়।

এটি শর্তের সাথে মেলা প্রথম উপাদানটি (একটি মাত্র) রিটার্ন করে।

querySelectorAll:

এটিও অনেকটা querySelector-এর মতোই CSS সিলেক্টর ব্যবহার করে কাজ করে।

এটি শর্তের সাথে মেলা একাধিক উপাদানকে ধরতে সহায়তা করে।

এটি মূলত একটি NodeList রিটার্ন করে।

এতে সরাসরি forEach লুপ ব্যবহার করা যায়।

এটি স্ট্যাটিক।

3. What is Event Bubbling? And how does it work?

Ans: Event Bubbling হলো Dom ar একটি প্রাকৃতিক প্রক্রিয়া। এখানে আমি যদি কোনো চাইল্ড এলিমেন্টে ক্লিক করি, তখন সেই ইভেন্টটি বুদবুদের (bubble) মতো তার প্যারেন্টের কাছে যায়। এভাবে এটি ধাপে ধাপে একদম window পর্যন্ত পৌঁছায়। অর্থাৎ, একটি বাটনে ক্লিক করলে সেটি প্রথমে বাটনের প্যারেন্টের কাছে যাবে, তারপর তার গ্র্যান্ডপ্যারেন্টের কাছে যাবে। সে যখনই কোনো প্যারেন্টের কাছে যায়, তখন দেখে যে সেই প্যারেন্টের কোনো ক্লিক ইভেন্ট আছে কিনা—থাকলে সেটিও এক্সিকিউট (চালু) করে।

4. What is Event Delegation in JavaScript? Why is it useful?

Ans: এটি এমন একটি কৌশল যা মূলত Event Bubbling-কে কাজে লাগিয়ে অনেকগুলো ক্লিক বা কাজ একসাথে হ্যান্ডেল করে। যেমন—প্যারেন্ট এলিমেন্টে একটি ক্লিক ইভেন্ট বসিয়ে তার সব চিলড্রেন বা সন্তানদের ক্লিক অটোমেটিক হ্যান্ডেল করা যায়। এর ফলে প্রতিটি এলিমেন্টে আলাদা আলাদা করে ইভেন্ট লিসেনার যোগ করার প্রয়োজন পড়ে না।

5. What is the difference between preventDefault() and stopPropagation() methods?
Ans:
preventDefault():
এটি মূলত HTML এলিমেন্টের কিছু ডিফল্ট আচরণ (default behavior) পরিবর্তন করার জন্য ব্যবহার করা হয়। যেমন—ফর্ম সাবমিট করলে পেজ রিলোড হওয়া বন্ধ করতে এটি ব্যবহার করা হয়। আবার লিঙ্কে (<a> ট্যাগ) ক্লিক করলে অন্য পেজে চলে যাওয়া বন্ধ করতেও এটি ব্যবহৃত হয়।

stopPropagation():
এটি ব্যবহার করা হয় Event Bubbling বন্ধ করার জন্য। যদি আমরা চাই ইভেন্টটি তার প্যারেন্টের কাছে না পৌঁছাক, তবে আমরা এই মেথডটি ব্যবহার করি।
